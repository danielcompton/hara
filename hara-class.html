class

functions for reasoning about classes

1    Introduction

hara.class are a set of functions that give information about class properties. There used by hara.reflect and hara.object for their reflection calls.

1.1    Installation

Add to project.clj dependencies:

[im.chit/hara.class "2.3.1"]

Individual namespaces can be added seperately:

[im.chit/hara.class.checks "2.3.1"]
[im.chit/hara.class.inheritance "2.3.1"]

2    API - checks

hara.class.checks contain class checking predicates:

API

abstract?  dispatches?  interface?  multimethod?  protocol?  

abstract? ▴

"Returns `true` if `class` is an abstract class" (abstract? java.util.Map) => true (abstract? Class) => false

dispatches? ▴

"Returns `true` if the multimethod contains a value for dispatch" (dispatches? print-method Class) => true

interface? ▴

"Returns `true` if `class` is an interface" (interface? java.util.Map) => true (interface? Class) => false

multimethod? ▴

"Returns `true` if `obj` is a multimethod" (multimethod? print-method) => true (multimethod? println) => false

protocol? ▴

"Returns `true` if `obj` is a protocol" (defprotocol ISomeProtocol) (protocol? ISomeProtocol) => true (protocol? clojure.lang.ILookup) => false

3    API - inheritance

hara.class.inheritance contain inheritance checking predicates:

API

ancestor-list  ancestor-tree  best-match  

ancestor-list ▴

"Lists the direct ancestors of a class" (ancestor-list clojure.lang.PersistentHashMap) => [clojure.lang.PersistentHashMap clojure.lang.APersistentMap clojure.lang.AFn java.lang.Object]

ancestor-tree ▴

"Lists the hierarchy of bases and interfaces of a class." (ancestor-tree Class) => [[java.lang.Object #{java.io.Serializable java.lang.reflect.Type java.lang.reflect.AnnotatedElement java.lang.reflect.GenericDeclaration}]]

best-match ▴

"finds the best matching interface or class from a list of candidates" (best-match #{Object} Long) => Object (best-match #{String} Long) => nil (best-match #{Object Number} Long) => Number
