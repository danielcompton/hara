expression

interchange between code and data

1    Introduction

hara.expression allow transformations of forms into functions and predicates in a succint way.

1.1    Installation

Add to project.clj dependencies:

[im.chit/hara.expression "2.2.13"]

Individual namespaces can be added seperately:

[im.chit/hara.expression.compile "2.2.13"]
[im.chit/hara.expression.form "2.2.13"]
[im.chit/hara.expression.load "2.2.13"]
[im.chit/hara.expression.shorthand "2.2.13"]

2    API - compile

hara.expression.compile

API

applym  const  

applym ▴

"Allow macros to be applied to arguments just like functions" (applym const '((+ 1 2))) => 3 (macroexpand '(applym const '((+ 1 2)))) => 3

const ▴

"converts an expression into a constant at compile time" (const (+ 1 2)) => 3 (macroexpand '(const (+ 1 2))) => 3

3    API - form

hara.expression.form:

API

form-apply  form-eval  form-fn  form-prep  form-require  

form-apply ▴

"Applies a list as a function to an argument vector" (form-apply '(+ 1 %1 %2) [2 3]) => 6

form-eval ▴

"Evaluates a list as a functions and to a set of arguments." (form-eval '(+ 1 %1 %2) 2 3) => 6

form-fn ▴

"Creates a function out of a list" (let [my-inc (form-fn '(+ 1 %))] (my-inc 1) => 2 (meta my-inc) => {:source "#(+ 1 %)\n"})

form-prep ▴

"Prepares the form into a function form"

form-require ▴

"Makes sure that the namespace is loaded for a particular symbol" (form-require 'cons) => 'cons (form-require 'clojure.core/cons) => 'clojure.core/cons

5    API - shorthand

hara.expression.shorthand:

API

call->  check  check->  check-all  check?->  eq->  fn->  get->  shorthand-fn-expr  shorthand-form  

call-> ▴

"Indirect call, takes `obj` and a list containing either a function, a symbol representing the function or the symbol `?` and any additional arguments. Used for calling functions that have been stored as symbols." (call-> 1 '(+ 2 3 4)) => 10 (call-> 1 '(< 2)) => true (call-> {:a {:b 1}} '((get-in [:a :b]) = 1)) => true

check ▴

"checks" (check 2 2) => true (check 2 even?) => true (check 2 '(> 1)) => true (check {:a {:b 1}} '([:a :b] (= 1))) => true (check {:a {:b 1}} :a vector?) => false (check {:a {:b 1}} [:a :b] 1) => true

check-> ▴

"Shorthand ways of checking where `m` fits `prchk`" (check-> {:a 1} :a) => true (check-> {:a 1 :val 1} [:val 1]) => true (check-> {:a {:b 1}} [[:a :b] odd?]) => true

check-all ▴

"Returns `true` if `obj` satisfies all pairs of sel and chk" (check-all {:a {:b 1}} :a #(instance? clojure.lang.IPersistentMap %) [:a :b] 1) => true

check?-> ▴

"Tests obj using prchk and returns `obj` or `res` if true" (check?-> :3 even?) => nil (check?-> 3 even?) => nil (check?-> 2 even?) => true (check?-> {:id :1} '[:id (= :1)]) => true

eq-> ▴

"Compare if two vals are equal." (eq-> {:id 1 :a 1} {:id 1 :a 2} :id) => true (eq-> {:db {:id 1} :a 1} {:db {:id 1} :a 2} [:db :id]) => true

fn-> ▴

"Constructs a function from a form representation." ((fn-> '(+ 10)) 10) => 20

get-> ▴

"Provides a shorthand way of getting a return value. `sel` can be a function, a vector, or a value." (get-> {:a {:b {:c 1}}} :a) => {:b {:c 1}} (get-> {:a {:b {:c 1}}} [:a :b]) => {:c 1}

shorthand-fn-expr ▴

"Makes a function expression out of the form" (shorthand-fn-expr '(+ 2)) => '(fn [%] (+ % 2))

shorthand-form ▴

"Makes an expression using `sym`" (shorthand-form 'y '(str)) => '(str y) (shorthand-form 'x '((inc) (- 2) (+ 2))) => '(+ (- (inc x) 2) 2)
