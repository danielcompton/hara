io.environment

tools for versioning and environment

1    Introduction

hara.io.environment provides an easier interface for working with jvm properties and versioning.

1.1    Installation

Add to project.clj dependencies:

[im.chit/hara.io.environment "2.2.16"]

All functions are in the hara.io.environment namespace.

(require '[hara.io.environment :as env])

2    API

API

clojure-version  java-version  properties  require  version  

clojure-version ▴

"returns the current clojure version" (env/clojure-version) => (contains {:major anything, :minor anything, :incremental anything :qualifier anything})

java-version ▴

"returns the current java version" (env/java-version) => (contains {:major anything, :minor anything, :incremental anything :qualifier anything})

properties ▴

"returns jvm properties in a nested map for easy access" (->> (env/properties) :os) => (contains {:arch anything :name anything :version anything})

require ▴

"only attempts to load the files when the minimum versions have been met" (env/require {:java {:major 1 :minor 8} :clojure {:major 1 :minor 6}} '[hara.time.data.zone java-time-zoneid] '[hara.time.data.instant java-time-instant] '[hara.time.format java-time-format-datetimeformatter])

version ▴

"alternate way of getting clojure and java version" (env/version :clojure) => (env/clojure-version) (env/version :java) => (env/java-version)
